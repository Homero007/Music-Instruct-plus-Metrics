Ciclo principal Todo lo que aparece abajo alimenta el siguiente paso.

No hay herramientas aisladas: los datos se preparan, se transforman a tokens, entrenan un modelo, generan nuevas versiones y vuelven a audio para escuchar y comparar.

  1. AudioSeleccionar pistas y crear clips.
  2. RepresentaciónStems, MIDI, features y tokens.
  3. ModeloEntrenar con los tokens válidos.
  4. GeneraciónCrear varias versiones nuevas.
  5. Audio finalRenderizar, escuchar y descargar.
  6. ExportaciónDescargar tokens de entrada o salida.

¿Cómo elijo qué evaluar?

Para cada género escribe en “Canciones a evaluar” cuántas pistas generadas quieres comparar contra audio real del mismo género. El sistema elige automáticamente las de mejor puntaje, las reparte entre las corridas y empareja el mismo número de originales reales. No hace falta llenar nada corrida por corrida.

  • Usa “Seleccionar por género” (arriba) para llenar todos los géneros de una vez con la cantidad objetivo.
  • ¿Quieres control fino? Abre “Elegir manualmente por corrida” dentro de cada género.
  • Los géneros sin originales reales (p. ej. unknown, “Transformer normal” sin género) no se pueden comparar y quedan deshabilitados.

    Glosario del ciclo Qué produce cada etapa
    Clip

    Fragmento corto de audio usado como ejemplo de entrenamiento.

    Stem

    Capa separada del audio original: batería, bajo, voz u otros.

    MIDI por capas

    Notas editables separadas por función musical.

    Token

    Evento musical que el modelo puede aprender y volver a generar.

    Modelo

    Archivo entrenado con tokens para generar nueva música.

    Render

    Conversión del MIDI final a WAV/MP3 para escucharlo.

    Ranking

    Comparación de varias versiones generadas.

    Embedding

    Vector musical que permite fusionar o condicionar resultados.

    Exportación

    Descarga organizada de tokens de entrada o de salida.

    Comparativa Técnica de Generación Musical

    Proyecto de Investigación

    Este proyecto propone una comparativa técnica y estructural de modelos generativos diseñados para la creación de melodías musicales coherentes y perceptualmente consistentes a partir de descripciones textuales.

    Institución Instituto Politécnico Nacional
    Carrera Ingeniería en Inteligencia Artificial
    Géneros Demostrados Clásica, Reggae

    Objetivos Principales

    • Comparativa de Arquitecturas: Análisis estructural de modelos generativos musicales
    • Optimización con Monte Carlo: Integración de métodos probabilísticos
    • Validación Experta: Evaluación con músicos profesionales
    • Innovación en Métricas: Propuesta de evaluación en doble paso
